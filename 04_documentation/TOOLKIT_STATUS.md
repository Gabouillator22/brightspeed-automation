# Toolkit Status Matrix

This file is the mandatory cross-session execution ledger for Brightspeed
toolkit stabilization. It replaces chat memory for current command status.

## Status values

- `working` = tested and behaving as expected for the exercised case
- `partial` = usable but with known issues or incomplete coverage
- `broken` = known failing behavior
- `untested` = present in repo but not yet meaningfully exercised
- `regressed` = previously working behavior now fails

## Canonical workflow

The active deliverable is the loader-driven workflow in
`05_toolkit/lisp/bs_loader.lsp`, not the full historical command surface.
Compatibility aliases stay secondary until the canonical command they route to
is stable.

### Canonical handshakes

- Sheet planning: `bssheet_plan.csv` is the primary handoff.
- Sheet planning compatibility output: `bssheet_plan.lsp`.
- KMZ sheet workflow export: `bssheet_route_selected.txt`.
- Parcel extraction output: `parcels_<kmz_stem>.dxf`.

### Sample case catalog

These case IDs are the required repeatable validation targets for the matrix.

| Case ID | Scenario | Minimum setup |
|---|---|---|
| `SR-01` | Straight road baseline | One centerline with clean ROW/EOP expectations, accepted BORDER sheets, and simple fiber geometry. |
| `CR-01` | Curved/intersection road | Curved centerline or intersection with corner fillets, driveway crossings, and sheet bends. |
| `PC-01` | Parcel conflict case | Property lines crossing and running parallel to ROW/TRAP geometry. |
| `KMZ-01` | KMZ import case | KMZ with handholes, running lines, buried, aerial, and e-lash folders that map into the active layer names. |
| `AN-01` | Annotation case | Accepted sheets plus buried/aerial/structure source geometry ready for labels and callouts. |
| `FC-01` | Final permit cleanup case | Accepted BORDER sheets, cleanup limit, incoming linework, and QA labels ready for audit and trimming. |
